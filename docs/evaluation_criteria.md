# 평가 기준표

Generator와 Evaluator가 공유하는 품질 기준.

## 채점 항목

### 1. 기능 정확성 (비중: 40%)
SPEC의 각 기능이 올바르게 동작하는가?

합격: 모든 SPEC 기능 구현, 테스트 통과, 엣지 케이스 처리, 명확한 에러 메시지
불합격: SPEC 기능 누락, 테스트 실패, 예외 미처리, 잘못된 출력값

### 2. 코드 품질 (비중: 30%)
유지보수 가능하고 이해하기 쉬운 코드인가?

합격: 단일 책임 원칙 준수, 타입 힌트 완비, 의미 있는 변수/함수명, 금지 패턴 없음
불합격: 전역 변수, 빈 except, 하드코딩, 함수 비대, 타입 힌트 누락

### 3. 성능 (비중: 15%)
불필요한 비효율이 없는가?

합격: 적절한 자료구조 선택, 불필요한 중복 계산 없음, 메모리 누수 없음
불합격: 회피 가능한 O(n²) 방치, 동일 데이터 반복 로드, 닫지 않는 리소스

### 4. 테스트 커버리지 (비중: 15%)
핵심 로직이 검증되어 있는가?

합격: 각 기능마다 정상/엣지 케이스 테스트, pytest 전체 통과, 경계값 테스트 포함
불합격: 테스트 파일 없음, 정상 케이스만 있음, 테스트 실패 존재

## 판정

가중 점수 = (정확성×0.4) + (코드품질×0.3) + (성능×0.15) + (테스트×0.15)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격 (피드백 반영 후 재검수)
- 5.0 미만 → 불합격
- 기능 정확성 또는 테스트 커버리지 4점 이하 → 무조건 불합격

## 피드백 형식

```
**전체 판정**: [합격 / 조건부 합격 / 불합격]
**가중 점수**: X.X / 10.0

**항목별 점수**:
- 기능 정확성: X/10 — [한 줄 코멘트]
- 코드 품질: X/10 — [한 줄 코멸트]
- 성능: X/10 — [한 줄 코멘트]
- 테스트 커버리지: X/10 — [한 줄 코멘트]

**SPEC 기능 체크**:
- [PASS/FAIL] 기능 1: [설명]
- [PASS/FAIL] 기능 2: [설명]

**테스트 실행 결과**:
[pytest 출력 요약]

**구체적 개선 지시**:
1. [파일명:함수명 — 무엇을 어떻게 고칠 것]
2. [파일명:함수명 — 무엇을 어떻게 고칠 것]

**방향 판단**: [현재 방향 유지] 또는 [완전히 다른 접근 시도]
```
